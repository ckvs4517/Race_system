# Spin League

戰鬥陀螺賽事管理 Web 系統。這是一個不需要建置工具即可部署的前端 MVP，使用瀏覽器原生 ES Modules，支援桌面與手機版介面。

## 目前功能

- 現代深色競技場介面與響應式版面。
- 獨立記分板：加分、減分、重設、復原與交換選手。
- 建立單淘汰賽：支援 2 至 32 位參賽者並自動處理輪空晉級。
- 賽事生命週期：新賽事先保持「準備中」，可編輯名稱與名單；確認開始後才鎖定賽程並開放記分。
- 賽程表：依參賽人數產生完整比賽輪次，可從待比賽節點進入專用記分板。
- 正式比賽結果：保存比分與勝者，自動更新下一輪參賽者並產生冠軍。
- 賽事管理：可從賽事列表刪除賽事，刪除前會再次確認。
- 使用瀏覽器 `localStorage` 保存賽事資料，並相容第一版的既有資料。

## 專案架構

```text
src/
├─ core/       # 路由等應用程式基礎功能
├─ data/       # localStorage 與狀態存取
├─ domain/     # 賽事資料模型與淘汰賽邏輯
├─ ui/         # 共用介面、圖示與網站框架
├─ views/      # 首頁、記分板、賽程與管理頁面
├─ styles/     # 全站視覺與響應式樣式
└─ main.js     # 應用程式進入點與事件協調
```

資料層與賽事邏輯不依賴畫面實作，後續可替換成 Supabase 等雲端服務，或把 UI 遷移到 React／Vue。

## 本地預覽

瀏覽器需要透過 HTTP 載入 ES Modules，建議使用 VS Code Live Server 開啟專案目錄。部署到 GitHub Pages 或 ChatGPT Sites 時可直接載入。

## 測試

瀏覽器開啟 `tests/tournament.test.html` 可驗證 32 人賽程、準備中編輯、開始後鎖定、輪空、晉級、比分與舊資料轉換邏輯。測試頁顯示 `PASS` 即代表通過。
