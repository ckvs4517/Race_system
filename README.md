# Spin League

戰鬥陀螺賽事管理 Web 系統，提供公開賽程、獨立記分板、主辦方後台、單淘汰賽、瑞士制與多戰鬥台排程。

公開網站：https://spin-league-tournament.ckvs4517.chatgpt.site

## 目前定位

目前版本是一套支援多戰鬥台裁判同步的雲端賽事管理系統。正式賽事資料保存在 Cloudflare D1，不再使用瀏覽器 `localStorage`。

不同裁判可以同時輸入不同場比賽；系統會利用賽事版本檢查阻止舊資料覆蓋新結果，發生衝突時先取得最新資料並自動合併重試。同一場被重複修改時則會停止送出並提示裁判確認。

## 已完成功能

- 公開首頁、賽事列表、賽程與排行榜。
- 獨立記分板：加分、減分、復原、重設及交換選手。
- PIN 保護的主辦方後台。
- 建立、編輯、複製及刪除賽事。
- 2 至 32 位參賽者。
- 建立時自動隨機排列參賽者，開始前可重新隨機分組。
- 準備中、進行中與已完成的賽事生命週期。
- 單淘汰賽：首輪隨機種子、後續表現種子、輪空、晉級與冠軍。
- 瑞士制：依戰績配對、優先避免重複對手、輪空分配及指定輪數。
- 瑞士制分組顯示：第二輪顯示勝者組／敗者組，後續依勝場分組。
- 瑞士制排名：勝場、對手分（Buchholz）、得失分差及總得分。
- 重新比賽：清除該場及受影響的後續輪次，重新計算統計與配對。
- 1 至 8 台戰鬥台，每輪對戰平均分配到各台顯示。
- 雲端 D1 資料庫保存正式賽事。
- 單一賽事更新、版本衝突保護與不同場賽果自動合併。
- 公開賽程及非記分中的後台頁面每 3 秒自動檢查最新賽果。
- JSON 完整備份與還原。
- 賽事總覽 CSV 與對戰明細 CSV；明細包含戰鬥台資訊。
- 桌面與手機響應式介面及手機效能最佳化。
- 舊版賽事資料相容處理。

## 使用模式

### 公開模式

- 不需要登入。
- 可查看賽事、每輪對戰、比分、冠軍與排行榜。
- 可使用不影響正式賽事的獨立記分板。
- 不能建立賽事或修改正式比分。

### 主辦方模式

- 使用後端驗證的共用 PIN 登入。
- 可建立與管理賽事、開始賽事、輸入比分及重新比賽。
- 可進入資料管理頁匯出、備份及還原資料。

## 賽制

### 單淘汰賽

- 輸掉一場即淘汰，勝者晉級直到產生冠軍。
- 奇數首輪隨機選出一位種子輪空。
- 後續奇數輪依平均得分、得失分差與較少輪空次數選出表現種子。

### 瑞士制

- 2–4 人進行 2 輪。
- 5–8 人進行 3 輪。
- 9–16 人進行 4 輪。
- 17–32 人進行 5 輪。
- 配對優先選擇勝場相同且尚未交手的選手。
- 奇數人時，優先讓排名較低且尚未輪空的選手輪空。
- 最終依勝場、對手分、得失分差及總得分排序。

## 戰鬥台

建立或編輯準備中的賽事時，可設定 1 至 8 台戰鬥台。系統會依比賽編號將每輪對戰平均分配到「戰鬥台 1、戰鬥台 2」等區塊；不同戰鬥台的裁判可以使用不同裝置同步送出各自負責的比賽結果。

## 資料與架構

```text
src/
├─ core/       # 路由
├─ data/       # 前端狀態與雲端 API 存取
├─ domain/     # 賽事模型與共用流程
├─ formats/    # 單淘汰賽與瑞士制策略
├─ ui/         # 共用網站框架與圖示
├─ views/      # 首頁、記分板、賽程、後台與資料管理
├─ styles/     # 全站視覺與響應式樣式
└─ main.js     # 應用程式進入點與事件協調

worker/
└─ index.js    # Cloudflare Worker API、PIN 驗證與 D1 存取
```

- 前端使用瀏覽器原生 ES Modules。
- 後端使用 Cloudflare Worker。
- 正式賽事以 JSON 儲存在 Cloudflare D1。
- 管理權杖只保存在當次瀏覽器工作階段的 `sessionStorage`。
- 網站由 ChatGPT Sites 建置及公開部署。

## 同步與已知限制

- 一般修改只更新單一賽事，不再以整批資料覆蓋所有賽事。
- 每場賽事具有獨立版本編號；後端只接受基於最新版本的修改。
- 不同場比賽同時送出若發生衝突，前端會載入最新版本、保留其他裁判的結果，再自動套用目前比分重試一次。
- 同一場比賽已由其他裁判完成時，不會覆蓋既有賽果，畫面會回到最新賽程並顯示提示。
- 所有非記分中的頁面每 3 秒檢查一次雲端版本；正式記分畫面暫停自動刷新，避免裁判輸入中的分數被重設，送出時仍會執行版本檢查。
- JSON 全量還原仍屬刻意取代全部資料的管理操作，執行前會顯示確認警告。
- 目前仍使用共用主辦方 PIN，尚未區分裁判身分、指定戰鬥台權限與操作紀錄。
- 目前採短間隔自動更新，尚未使用 WebSocket 即時推播。

## 建置

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\build.ps1
```
